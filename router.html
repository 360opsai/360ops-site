360router

Smart AI Router — Local First, Cloud When Needed

Route your AI requests intelligently between local models (Ollama, LM Studio, vLLM) and cloud providers (Anthropic, OpenAI, Groq, Gemini). Privacy-aware. Open source. Deterministic routing with the Rubik 5-move algorithm.

Core Capabilities

Built for Privacy
and Performance

360router makes intelligent routing decisions based on model capability, cost, latency, and data sensitivity.

🏠
Local First

Tries local models before cloud. Data stays on your machine. Only sends to cloud when local models can't handle it.

🎯
Smart Routing

Rubik 5-move algorithm picks the cheapest capable model. No token waste. No manual selection.

⚡
Circuit Breaker

Dead endpoints skipped instantly. Auto-recovery when services come back online. No failed requests.

🔒
Privacy Aware

Detects sensitive data being sent to cloud. Routes to local models or blocks the request entirely.

How It Works

Interactive Setup.
Zero Config Overhead.

360router discovers your local models, connects to cloud providers, and starts routing intelligently.

$ 360router init
Scanning localhost... Found Ollama (3 models)
Add cloud providers? [Anthropic, OpenAI, Groq]
Config saved.
$ 360router status
Ollama (local) — online, 12ms, 3 models
Anthropic (cloud) — online, 340ms
$ 360router route "Summarize this contract"
Routed to: Ollama llama3.3:8b (local, 0.8s)
Reason: medium complexity, local capable
